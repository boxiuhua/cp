# 双色球真实历史开奖数据接入 — 设计文档

日期:2026-07-04
项目:`lottery_stats`(彩票随机性验证工具,Rust,零外部依赖)

## 背景与目标

现有工具用**模拟随机数**跑 4 项统计检验(卡方均匀性、赌徒谬误、游程独立性、蒙特卡洛资金曲线),从数学上论证彩票不可预测。本次扩展目标:**接入真实双色球历史开奖数据**,在真实数据上复现同一套结论,并新增"预测打脸"实验,实证任何选号策略都不优于随机。

范围策略:**先 A 后 B** —— 先把双色球一种彩票的完整链路(读文件 → 真实数据检验 → 预测实验)跑通,验证设计;通用多彩种格式留待后续。

零依赖 / 离线可复现的设计哲学保持不变。

## ① 文件格式 `data/ssq.csv`

```
# 双色球历史开奖数据  期号,日期,红1,红2,红3,红4,红5,红6,蓝
# 红球 6 个(1-33 互不相同),蓝球 1 个(1-16)。# 开头为注释,空行忽略。
2024001,2024-01-02,1,7,15,22,28,33,9
2024002,2024-01-04,3,5,11,19,26,31,2
```

规则:
- 每行 9 个字段:`期号,日期,红1..红6,蓝`。
- 首行可为表头(第一字段非纯数字时自动识别并跳过);`#` 开头的行为注释;空行忽略。
- **校验**:红球恰好 6 个,均 ∈ [1,33] 且互不相同;蓝球 1 个 ∈ [1,16]。
- **容错**:不合规的行跳过并记录 `(行号, 原因)`,最后汇总"成功解析 N 期,跳过 M 行(附原因列表)"。解析不因单行错误中断。
- **文件缺失**:优雅降级 —— 打印格式说明与预期路径,跳过真实数据部分,模拟部分照常运行。
- 交付时放入若干**占位示例行**(明确标注"示例数据,请替换为真实开奖"),并附 `data/README.md` 说明格式与官方数据来源;真实数据由用户后续填充。

## ② 真实数据上的分析

对加载得到的 `Vec<Draw>` 依次执行,并与模拟结果**并排对照**:

1. **卡方均匀性**:统计 33 个红球在真实历史中的出现频次,计算 χ² 与右尾 p 值。真随机 → p 不显著 → 无法拒绝均匀分布。报告注明真实期数(数千)远少于模拟(百万),统计功效较低。
2. **冷热号分析**:列出真实数据中最冷/最热红球及频次差异百分比,并给出理论标准差作参照,说明该差异在当前样本量下仍属随机涨落。
3. **赌徒谬误**:选一个红球,统计"上期没出 → 本期出"的条件概率,对比无条件概率 6/33,验证二者 ≈ 相等。
4. **游程检验**:把某红球"逐期是否出现"编码为 0/1 序列,做 Wald–Wolfowitz 游程检验,验证序列独立(无自相关)。

## ③ 预测"打脸"实验(核心新增)

按时间顺序遍历历史。从第 W 期(默认窗口 W=30)起,用前 W 期走势预测下一期的 6 个红球,与真实开出号码比对,统计命中个数(0–6)。三种策略同台竞技:

- **冷号策略**:选窗口内出现最少的 6 个号(赌"冷号回补")。
- **热号策略**:选窗口内出现最多的 6 个号(赌"热号延续")。
- **随机基线**:随机选 6 个号。

指标:各策略的平均命中数。纯运气的理论期望 = 6 × 6/33 ≈ **1.09 个/期**。预期三种策略均值都落在 1.09 附近且彼此无显著差异 → 实证选号策略不优于瞎蒙。报告给出三者均值,以及"与随机基线的差异是否显著"的简单判断。

平局/并列取号规则:频次相同时按号码升序取前若干个,保证结果可复现。

## ④ 代码结构与测试

### 模块划分

- `src/main.rs` — 编排与打印;保留纯数学函数(`comb`、`chi2_pvalue`、`gammln`、`gammp_series`、`gammq_cf`、`erf`、`normal_two_sided_p`)、`Rng`、彩种定义、概率/期望表、既有 4 项模拟演示。新增两个**共享函数**供模拟与真实路径复用:
  - `chi2_from_freq(freq: &[u64], expected: f64) -> f64`(频次数组 → χ²)
  - `runs_z(seq: &[bool]) -> (f64 /*runs*/, f64 /*mu*/, f64 /*z*/)`(0/1 序列 → 游程 Z)
  既有 `chi_square_uniformity_demo` 与 `runs_test_demo` 改为调用这两个共享函数。
- `src/ssq.rs` — 新模块:
  - `struct Draw { issue: String, date: String, reds: [u8; 6], blue: u8 }`
  - `struct SkipInfo { line: usize, reason: String }`
  - `fn load_ssq(path: &str) -> Result<(Vec<Draw>, Vec<SkipInfo>), String>`
  - 真实数据分析函数(接收 `&[Draw]`,并接收 `main` 传入的共享数学函数结果或 `Rng`):卡方、冷热、赌徒谬误、游程、预测实验。

`main.rs` 通过 `mod ssq;` 引入,并把需要的纯函数以 `pub` 暴露给 `ssq`(或将共享数学函数一并放入可被 `ssq` 调用的位置)。

### 单元测试(`#[cfg(test)]`)

- `load_ssq`:正常行解析;跳过注释、空行、表头;非法行被拒收(红球越界、红球重复、红球个数≠6、蓝球越界、字段数不对、非数字)。
- 校验器:合法输入通过,各类非法输入返回对应错误。
- `chi2_from_freq`:完全均匀频次 → χ² ≈ 0;已知偏斜输入 → 已知 χ² 值。
- `runs_z`:构造已知序列核对 runs 计数与 μ。
- 预测实验:构造"红球 7 每期必出"的人造数据集,验证热号策略命中显著偏高 —— 反向确认实验逻辑正确(真随机下才回归 ≈1.09)。

```
[1-2] 概率表 / 期望值              (不变)
[3-6] 模拟随机数的 4 项检验         (不变)
[7]   真实数据篇:
      - 加载摘要(解析 N 期 / 跳过 M 行)
      - 卡方 / 冷热 / 赌徒谬误 / 游程   ← 模拟 vs 真实 并排
      - 预测打脸实验(冷 / 热 / 随机 三策略命中率)
[总结] 增补一条:真实历史数据与理论 / 模拟结论一致
```
